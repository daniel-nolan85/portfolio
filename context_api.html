React Context API

Exploring how to manage state using React's Context API



State is a JavaScript object used in React to store data that can be used in components. State can be passed down between components through props but only in one direction, from parent to child.


This diagram represents the DOM of an application and shows which components the Header is able to pass data to via props, and which ones it isn't.


So, what would we do if we needed to pass data from the Header to say, the Article component? Well, the obvious answer might be to simply 'lift the state up' from the Header to the App component, since all the components are children of the App. This could work, however it would be likely to involve a process known as 'prop drilling', which is where state has to be passed through components that don't actually need it, just so it is able to flow through to the ones that do. For example, to pass state from App to Article in this way, it would also need to make its way through Main. In larger projects this can easily become unmanageable and can also affect performance of the application.


A number of exterior libraries, such as Redux and MobX, have been developed for React that allow for more state management control, however the problem can also be addressed without the need to install any additional packages, using React's built-in Context API.


Let's take this simple Navbar component for example:


const Navbar = () => {
  return (
    <header>
      <nav>
        <Link to='#home'>Home</Link>
        <Link to='#about'>About</Link>
        <Link to='#contact'>Contact</Link>
      </nav>
      <button>Login</button>
    </header>
  );
};

export default Navbar;


Now imagine we want the button to display a modal with a form to allow the user to log in. We might decide to apply the useState hook to display a Login component depending on a Boolean value, like this:


import React, { useState } from 'react';

const Navbar = () => {
  const [showLogin, setShowLogin] = useState(false);

  return (
    <header>
      <nav>
        <Link to='#home'>Home</Link>
        <Link to='#about'>About</Link>
        <Link to='#contact'>Contact</Link>
      <nav>
      <button onClick={() => setShowLogin(true)}>Login</button>
    <header>
  );
};

export default Navbar;


Sure, this is fine, but assume the Login component also needs access to this state variable, here's where the Context API comes into play.


We could start by creating a new folder called context, and a new file inside it called index.js


In this file we import the createContext hook from React and save it as a variable. We then create a new component that makes use of useState to store the data we require.


import React, { useState, createContext } from 'react';

export const LoginContext = createContext();

export const LoginProvider = (props) => {
  const [showLogin, setShowLogin] = useState(false);
};


createContext has a built-in Provider method which we can chain to our variable and give it our state as a prop, value. We then return and wrap this around props.children to allow all child elements access to use it:


return (
  <LoginContext.Provider value={[showLogin,setShowLogin]}>
    {props.children}
  </LoginContext.Provider>
);


In full our index.js will display like this:


import React, { useState, createContext } from 'react';

export const LoginContext = createContext();

export const LoginProvider = (props) => {
  const [showLogin, setShowLogin] = useState(false);

  return (
    <LoginContext.Provider value={[showLogin,setShowLogin]}>
      {props.children}
    </LoginContext.Provider>
  );
};


We need to then import our component into the root of our project in App.js, and wrap it around all components that need access to the state.


import Login from './components/Login';
import Navbar from './components/Navbar';
import { LoginProvider } from './context';

const App = () => {
  return (
    <LoginProvider>
      <Navbar />
      <Login />
    </LoginProvider>
  );
};

export default App;


It's then just a simple case of importing LoginContext into any component that needs it, in our case Login.js and Navbar.js, and provide it with the useContext hook similar to initializing state:


import React, { useContext } from 'react';
import { LoginContext } from '../context';

const Login = () => {
  const [showLogin, setShowLogin] = useContext(LoginContext);
  ...
};

export default Login;


And that's it. Happy days!
